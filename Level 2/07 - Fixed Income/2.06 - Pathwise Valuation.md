# 2.06 — Pathwise Valuation

> [!abstract] Learning Outcome
> Describe pathwise [[Valuation|valuation]] in a binomial [[Interest rate|interest rate]] framework and calculate the value of a fixed-income instrument given its cash flows along each path.

**Parent:** [[Module 2 - Arbitrage-Free Valuation Framework]]

## What Is Pathwise Valuation?

> [!tip] Analogy
> [[Backward Induction|Backward induction]] starts at the office and works backward through every intersection. Pathwise [[Valuation|valuation]] simply **drives every possible route** start-to-finish, times each one, then averages them.

Instead of working backward node-by-node, pathwise [[Valuation|valuation]]:

1. **Lists** every possible [[Interest rate|interest rate]] path through the tree
2. **Discounts** all cash flows along each path using that path's specific rate sequence
3. **Averages** the present values across all paths

---

## How Many Paths?

For an $n$-period tree: **$2^{n-1}$ unique paths**, each with [[Probability|probability]] $\left(\frac{1}{2}\right)^{n-1}$.

| Periods | Paths | [[Probability|Probability]] per path |
|:-------:|:-----:|:-------------------:|
| 2 | 2 | 50% |
| 3 | 4 | 25% |
| 4 | 8 | 12.5% |
| 10 | 512 | 0.195% |

---

## The Discounting Formula Along Each Path

$$V_{\text{path}} = \frac{C}{(1+r_0)} + \frac{C}{(1+r_0)(1+r_1)} + \frac{C + \text{Par}}{(1+r_0)(1+r_1)(1+r_2)}$$

Each cash flow is discounted by the **cumulative product** of all 1-period rates from $t=0$ to that cash flow's arrival date. The rates along each path act like a chain of forward rates.

---

## The 4 Paths Through Our 3-Year Tree

Bond: 3-year, 4.25% coupon, par = 100. Tree from [[2.04 - Calibrating the Binomial Tree]].

| Path | Direction | $r_0$ | $r_1$ | $r_2$ |
|:----:|:---------:|:-----:|:------:|:------:|
| 1 | Up → Up | 2.500% | 3.8695% | 5.5259% |
| 2 | Up → Down | 2.500% | 3.8695% | 4.5242% |
| 3 | Down → Up | 2.500% | 3.1681% | 4.5242% |
| 4 | Down → Down | 2.500% | 3.1681% | 3.7041% |

> [!warning] Paths 2 and 3 reach the same $t=2$ rate (4.5242%)
> But they pass through **different** $t=1$ rates (3.8695% vs. 3.1681%), so their present values differ. The path matters, not just the endpoint.

---

## Path-by-Path Calculation

### Path 1: Up-Up (highest rates → lowest value)

$$PV_1 = \frac{4.25}{1.025} = 4.1463$$

$$PV_2 = \frac{4.25}{(1.025)(1.038695)} = \frac{4.25}{1.06466} = 3.9921$$

$$PV_3 = \frac{104.25}{(1.025)(1.038695)(1.055259)} = \frac{104.25}{1.12352} = 92.7917$$

$$\text{Path 1 value} = 4.1463 + 3.9921 + 92.7917 = \mathbf{100.9301}$$

### Path 2: Up-Down

$$PV_3 = \frac{104.25}{(1.025)(1.038695)(1.045242)} = \frac{104.25}{1.11290} = 93.6767$$

$$\text{Path 2 value} = 4.1463 + 3.9921 + 93.6767 = \mathbf{101.8151}$$

### Path 3: Down-Up

$$PV_2 = \frac{4.25}{(1.025)(1.031681)} = \frac{4.25}{1.05747} = 4.0190$$

$$PV_3 = \frac{104.25}{(1.025)(1.031681)(1.045242)} = \frac{104.25}{1.10575} = 94.2799$$

$$\text{Path 3 value} = 4.1463 + 4.0190 + 94.2799 = \mathbf{102.4452}$$

### Path 4: Down-Down (lowest rates → highest value)

$$PV_3 = \frac{104.25}{(1.025)(1.031681)(1.037041)} = \frac{104.25}{1.09632} = 95.0904$$

$$\text{Path 4 value} = 4.1463 + 4.0190 + 95.0904 = \mathbf{103.2557}$$

---

## Final Answer

$$V = \frac{1}{4}(100.9301 + 101.8151 + 102.4452 + 103.2557)$$

$$\boxed{V = 102.1115 \approx 102.1139}$$

> [!success] Same as [[Backward Induction|backward induction]] and [[Spot rates|spot rates]]!
> The tiny difference (102.1115 vs. 102.1139) is rounding in intermediate steps.

---

## Pattern: Higher Rates → Lower Value

| Path | Rate Environment | Bond Value |
|:----:|:---------------:|:----------:|
| 1 (UU) | Highest rates | **100.93** (lowest) |
| 2 (UD) | Mixed-high | 101.82 |
| 3 (DU) | Mixed-low | 102.45 |
| 4 (DD) | Lowest rates | **103.26** (highest) |

This confirms the fundamental inverse relationship between rates and bond prices.

---

## Three Methods Compared

| Method | Value | Approach |
|--------|:-----:|----------|
| [[Spot rates|Spot rates]] | 102.1139 | Discount each CF at maturity-matched [[Spot rate|spot rate]] |
| [[2.05 - Backward Induction and Option-Free Bonds\|Backward induction]] | 102.1139 | Work backward node-by-node through tree |
| **Pathwise** | 102.1115 | Discount along each path, then average |

### When to Use Each

| Bond Type | [[Spot rates|Spot Rates]] | [[Backward Induction|Backward Induction]] | Pathwise |
|-----------|:----------:|:------------------:|:--------:|
| Option-free | ✅ (simplest) | ✅ | ✅ |
| Callable/Putable | ❌ | ✅ (preferred) | ❌* |
| Path-[[Dependent|dependent]] (MBS) | ❌ | Tricky | ✅ |

> [!warning] *Why can't pathwise handle callable/[[Putable bonds|putable bonds]]?
> The call/put decision at a node requires knowing the bond's **value** at that node — which needs [[Backward Induction|backward induction]]. You can't know if a bond gets called along a path without first knowing its value at each node (circular logic).
> 
> However, pathwise [[Valuation|valuation]] excels for **path-[[Dependent|dependent]]** securities where cash flows depend on the specific path history. See [[2.07 - Monte Carlo Method]].

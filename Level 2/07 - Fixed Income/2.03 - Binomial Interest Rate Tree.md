# 2.03 — Binomial Interest Rate Tree

**Parent:** [[Module 2 - Arbitrage-Free Valuation Framework]]

## Why Do We Need a Tree?

The [[Module 2 - Arbitrage-Free Valuation Framework|spot-rate method]] works for **option-free** bonds. But bonds with **[[Embedded options|embedded options]]** (callable, putable) can be called or put depending on where rates go. We need to model **many possible rate paths** to figure out when the option kicks in.

> [!tip] Analogy
> Think of interest rates like a weather forecast. Tomorrow could be warmer or cooler. The day after, each outcome branches again. A **[[Binomial tree|binomial tree]]** is this branching forecast — but for interest rates.

---

## Key Assumptions

| Assumption | Why It Matters |
|------------|---------------|
| **Lognormal distribution** | Rates can't go negative: $r = e^{\ln(r)} > 0$ always |
| **Constant [[Volatility|volatility]]** ($\sigma$) | Simplifies the tree — same spread ratio at every node |
| **Recombining tree** | Up-then-down = down-then-up → computationally efficient |
| **Equal [[Probability|probability]]** (50/50) | Risk-neutral framework — real-world probabilities baked into rates |

---

## How Rates Branch

At each node, rates move **up** or **down** with equal 50% [[Probability|probability]]:

```
                    ┌─── R_u (upper rate)    [50%]
    r_today ────────┤
                    └─── R_d (lower rate)    [50%]
```

> [!important] Risk-Neutral Probability
> The 50/50 split is NOT a prediction about which direction is "more likely." It's a **risk-neutral** probability — the market's expectations are already embedded in the rate levels themselves.

---

## The $R_u = R_d \times e^{2\sigma\sqrt{t}}$ Relationship

This is the **central formula** of the binomial tree framework.

$$\boxed{R_u = R_d \times e^{2\sigma\sqrt{t}}}$$

### What Each Variable Means

| Variable | Definition | Role |
|----------|-----------|------|
| $R_u$ | Upper rate at the next node | Derived from $R_d$ |
| $R_d$ | Lower rate at the next node | The "anchor" — found by calibration |
| $\sigma$ | Assumed interest rate volatility (annualized) | INPUT — not solved for |
| $t$ | Length of each time step in years | Usually $t = 1$ for annual trees |

### Derivation from the Lognormal Assumption

**Step 1:** Since rates are lognormally distributed, $\ln(r)$ is normally distributed.

**Step 2:** In a binomial model, the up and down moves in log space are symmetric around a center, each separated by one standard deviation:

$$\ln(R_u) = \text{center} + \sigma\sqrt{t}$$
$$\ln(R_d) = \text{center} - \sigma\sqrt{t}$$

**Step 3:** The total gap in log space:

$$\ln(R_u) - \ln(R_d) = 2\sigma\sqrt{t}$$

**Step 4:** Using the log rule $\ln(A) - \ln(B) = \ln(A/B)$:

$$\ln\!\left(\frac{R_u}{R_d}\right) = 2\sigma\sqrt{t}$$

**Step 5:** Exponentiate both sides:

$$\frac{R_u}{R_d} = e^{2\sigma\sqrt{t}}$$

$$\therefore R_u = R_d \times e^{2\sigma\sqrt{t}}$$

> [!note] For Annual Trees ($t = 1$)
> When $t = 1$: $\sqrt{t} = 1$, so the formula simplifies to $R_u = R_d \times e^{2\sigma}$
> This is the form on the **Mark Meldrum formula sheet** (page 24).

### Why Multiplicative, Not Additive?

The lognormal assumption means rates move in **percentage** (multiplicative) terms, not absolute (additive) terms. The ratio $R_u / R_d = e^{2\sigma\sqrt{t}}$ is **constant across every node** in the tree — it doesn't matter where you are.

| Distribution | Spread | Negative Rates? |
|-------------|--------|----------------|
| Normal (Vasicek/Ho-Lee) | $R_u - R_d = 2\sigma\sqrt{t}$ (additive) | Yes |
| **Lognormal (KWF/Binomial tree)** | $R_u / R_d = e^{2\sigma\sqrt{t}}$ (multiplicative) | **No** |

## Numerical Example (Mark Meldrum Formula Sheet)

Given: $R_d = 3.1681\%$, $\sigma = 10\%$, $t = 1$

$$R_u = 3.1681\% \times e^{2(0.10)(1)} = 3.1681\% \times e^{0.20} = 3.1681\% \times 1.2214 = \mathbf{3.8695\%}$$

---

## The Recombining Property

```
t=0           t=1              t=2
                              R_uu = 5.5259%
             R_u = 3.8695%  ─┤
r₀ = 2.50% ─┤                R_ud = 4.5242%  ← SAME NODE
             R_d = 3.1681%  ─┤
                              R_dd = 3.7041%
```

The **up-then-down** path lands at the same node as **down-then-up**. This means:
- A 3-period tree has only **4** terminal nodes instead of $2^3 = 8$
- Computation stays manageable even for many periods
- At every time step $n$, there are $n+1$ nodes

## Connection to [[2.08 - Term Structure Models|Term Structure Models]]

The [[Binomial tree|binomial tree]] is the discrete implementation of the **Kalotay-Williams-Fabozzi (KWF)** model:

$$d\ln(r_t) = \theta_t\,dt + \sigma\,dz_t$$

The KWF model operates on $\ln(r)$ → lognormal rates → $R_u = R_d \times e^{2\sigma\sqrt{t}}$.

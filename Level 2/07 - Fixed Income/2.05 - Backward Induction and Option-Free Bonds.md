## What Is Backward Induction?

> [!tip] Analogy
> Imagine you're at the end of a corn maze and want to know how far the entrance is. You can't measure from the entrance forward — too many branches. Instead, start at the exit and work backward, one segment at a time.

Start at **maturity** (where everything is certain) and work backward through the tree, one time step at a time.

---

## The Formula

$$\boxed{V_{\text{node}} = \frac{C + 0.5 \times V_{\text{up}} + 0.5 \times V_{\text{down}}}{1 + r_{\text{node}}}}$$

### What Each Piece Means

| Component | Meaning |
|-----------|---------|
| $C$ | Coupon received at the end of this period (certain) |
| $0.5 \times V_{\text{up}} + 0.5 \times V_{\text{down}}$ | [[Probability|Probability]]-weighted average of bond's value in two future states |
| $r_{\text{node}}$ | The 1-period rate at THIS specific node (from the [[2.04 - Calibrating the Binomial Tree\|calibrated tree]]) |
| $V_{\text{node}}$ | The bond's value at this node — feeds into the node one step earlier |

---

## Full Worked Example

**Bond:** 3-year, 4.25% annual coupon, par = 100 ([[Straight bond|straight bond]] from MM formula sheet)

**Calibrated tree rates:** from [[2.04 - Calibrating the Binomial Tree#Calibrated Tree (Formula Sheet Values)]]

### Stage 1: Maturity Values ($t = 3$)

At all terminal nodes: $\text{coupon} + \text{par} = 4.25 + 100 = \mathbf{104.25}$

### Stage 2: Calculate $t = 2$ Values

At $t=2$, both branches lead to 104.25. The formula simplifies:

$$V = \frac{4.25 + 0.5(104.25) + 0.5(104.25)}{1 + r} = \frac{108.50}{1 + r}$$

| Node | Rate | Calculation | Value |
|:----:|:----:|:-----------:|:-----:|
| $uu$ | 5.5259% | $108.50 / 1.055259$ | **98.7909** |
| $ud$ | 4.5242% | $108.50 / 1.045242$ | **99.7377** |
| $dd$ | 3.7041% | $108.50 / 1.037041$ | **100.5264** |

### Stage 3: Calculate $t = 1$ Values

Now $V_{\text{up}}$ and $V_{\text{down}}$ differ — the tree is doing real work:

**Node $u$ (rate = 3.8695%):**

$$V = \frac{4.25 + 0.5(98.7909) + 0.5(99.7377)}{1.038695} = \frac{103.5143}{1.038695} = \mathbf{99.6580}$$

**Node $d$ (rate = 3.1681%):**

$$V = \frac{4.25 + 0.5(99.7377) + 0.5(100.5264)}{1.031681} = \frac{104.3821}{1.031681} = \mathbf{100.9215}$$

### Stage 4: Calculate $t = 0$ (Today's Price)

$$V_0 = \frac{4.25 + 0.5(99.6580) + 0.5(100.9215)}{1.025} = \frac{104.5398}{1.025} = \mathbf{102.1139}$$

> [!success] [[Arbitrage|Arbitrage]]-Free Value = **102.1139**
> This matches the Mark Meldrum formula sheet: Value of [[Straight bond|Straight Bond]] = 102.1139

## Comparison: Spot Rate Method

The same bond priced using [[Spot rates|spot rates]] ($z_1 = 2.5000\%$, $z_2 = 3.0080\%$, $z_3 = 3.5240\%$):

$$V = \frac{4.25}{(1.025)^1} + \frac{4.25}{(1.03008)^2} + \frac{104.25}{(1.03524)^3}$$

| Cash Flow | [[Spot rate|spot rate]] | Calculation | PV |
|:---------:|:---------:|:-----------:|:--:|
| CF₁ = 4.25 | $z_1$ = 2.500% | $4.25 / 1.025$ | 4.1463 |
| CF₂ = 4.25 | $z_2$ = 3.008% | $4.25 / 1.06106$ | 4.0053 |
| CF₃ = 104.25 | $z_3$ = 3.524% | $104.25 / 1.10946$ | 93.9623 |
| | | **Total** | **102.1139** |

> [!success] Same answer: **102.1139**

---

## Why Both Methods Give the Same Answer

| Feature | [[Spot rate|Spot Rate]] Method | [[Binomial tree|Binomial Tree]] |
|---------|-----------------|---------------|
| **Approach** | One direct path per CF | Many branching paths averaged |
| **Steps** | Single discount per CF | Step-by-step through nodes |
| **Source** | [[Spot curve|Spot curve]] | Calibrated to the same [[Par curve|par curve]] |
| **Result** | 102.1139 | 102.1139 |

Both methods derive from the **same [[Term structure|term structure]]** ([[Par curve|par curve]]). The tree was [[2.04 - Calibrating the Binomial Tree|calibrated]] to reproduce [[Benchmark|benchmark]] bond prices, which embed the same discount factors as the [[Spot curve|spot curve]]. They're two representations of the same [[Underlying|underlying]] [[Discount function|discount function]].

> [!important] When to Use Which
> 
> | Bond Type | Use |
> |-----------|-----|
> | **Option-free** | Either method ([[Spot rates|spot rates]] are simpler) |
> | **Callable / Putable** | **Must use the tree** — cash flows depend on rates at each node |
> 
> The [[Spot rate|spot rate]] method has no concept of "which path did rates take?" — it's blind to optionality. The tree knows every twist and turn.

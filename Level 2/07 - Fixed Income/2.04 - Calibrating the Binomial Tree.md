## What Is Calibration?

> [!tip] Analogy
> Calibration is like tuning a guitar. Each string needs to produce the right note. You twist the tuning peg ($R_d$) until the pitch (bond price) matches the target (par = 100).

**Calibration** = choosing the $R_d$ at each time step so the tree correctly prices known **[[On-the-run|on-the-run]] [[Benchmark|benchmark]] bonds** at par (100).

---

## What We Know Going In

| Input | Source | Role |
|-------|--------|------|
| [[Par curve|Par curve]] | Observable market yields | Target prices (each [[Benchmark|benchmark]] bond = 100) |
| [[Volatility|Volatility]] ($\sigma$) | Assumed | Fixed input — not solved for |
| $R_u = R_d \times e^{2\sigma\sqrt{t}}$ | [[2.03 - Binomial Interest Rate Tree]] | Once $R_d$ is known, $R_u$ follows automatically |

**What we solve for:** The $R_d$ at each time step.

> [!important] Why Par = 100?
> An [[On-the-run|on-the-run]] par bond, by definition, is priced at par. Its [[Coupon rate|coupon rate]] equals its yield. So a properly calibrated tree must produce a value of **exactly 100** for each [[Benchmark|benchmark]] bond.

---

## The Calibration Process

### Step 1: Year 1 — No Calibration Needed

The 1-year par rate IS the 1-year [[Spot rate|spot rate]]. No branching yet.

$$r_0 = \text{1-year par rate} = 2.5000\%$$

**Verify:** A 1-year par bond pays $102.50$ at $t=1$:

$$PV = \frac{102.50}{1.025} = 100.00 \checkmark$$

### Step 2: Year 2 — Iterative Calibration

Find $R_d$ at $t=1$ so the tree prices the **2-year par bond** (coupon = 3.00%) at exactly 100.

The **[[Backward Induction|backward induction]] formula** at each node:

$$V_{\text{node}} = \frac{C + 0.5 \times V_{\text{up}} + 0.5 \times V_{\text{down}}}{1 + r_{\text{node}}}$$

See [[2.05 - Backward Induction and Option-Free Bonds]] for full explanation of this formula.

**At $t=0$, we need:**

$$V_0 = \frac{C + 0.5 \times V_{1,u} + 0.5 \times V_{1,d}}{1 + r_0} = 100$$

---

## Worked Example: Calibrating $t=1$

Bond: 2-year, 3% coupon, par = 100. $\sigma = 10\%$, $r_0 = 2.50\%$.

### Guess 1: $R_d = 2.50\%$

$$R_u = 2.50\% \times e^{0.20} = 2.50\% \times 1.2214 = 3.0535\%$$

$$V_{1,u} = \frac{103}{1.030535} = 99.9481 \qquad V_{1,d} = \frac{103}{1.025} = 100.4878$$

$$V_0 = \frac{3.00 + 0.5(99.9481) + 0.5(100.4878)}{1.025} = \frac{103.218}{1.025} = 100.70$$

> [!warning] Too high! $R_d$ must be higher to push the price down.

### Guess 2: $R_d = 3.00\%$

$$R_u = 3.00\% \times 1.2214 = 3.6643\%$$

$$V_0 = \frac{3.00 + 0.5(99.3584) + 0.5(100.0000)}{1.025} = 100.17$$

> [!warning] Still too high.

### Converged Answer: $R_d = 3.1681\%$

$$R_u = 3.1681\% \times 1.2214 = \mathbf{3.8695\%}$$

$$V_{1,u} = \frac{103}{1.038695} = 99.1354 \qquad V_{1,d} = \frac{103}{1.031681} = 99.8370$$

$$V_0 = \frac{3.00 + 0.5(99.1354) + 0.5(99.8370)}{1.025} = \frac{102.4862}{1.025} = \mathbf{100.00} \checkmark$$

> [!success] Calibrated! These match the Mark Meldrum formula sheet values.

## Extending to Year 3

Repeat the identical process using the **3-year par bond** (coupon = 3.50%):

1. $r_0 = 2.50\%$, $R_{1,d} = 3.1681\%$, $R_{1,u} = 3.8695\%$ are already locked in
2. Guess $R_{2,dd}$ (the lowest rate at $t=2$)
3. Derive: $R_{2,ud} = R_{2,dd} \times e^{2\sigma}$ and $R_{2,uu} = R_{2,dd} \times e^{4\sigma}$
4. Work backward: $t=3 \to t=2 \to t=1 \to t=0$
5. Adjust $R_{2,dd}$ until tree price = 100

### Calibrated Tree (Formula Sheet Values)

| Time | Node | Rate |
|:----:|:----:|:----:|
| $t=0$ | $r_0$ | 2.5000% |
| $t=1$ | $R_{1,u}$ | 3.8695% |
| $t=1$ | $R_{1,d}$ | 3.1681% |
| $t=2$ | $R_{2,uu}$ | 5.5259% |
| $t=2$ | $R_{2,ud}$ | 4.5242% |
| $t=2$ | $R_{2,dd}$ | 3.7041% |

## Key Insight

> [!important] One Unknown Per Time Step
> Because of the relationship $R_u = R_d \times e^{2\sigma\sqrt{t}}$, there is only **one** unknown at each time step ($R_d$). All other rates at that time step are derived from it. This makes the iterative search a one-variable problem — computationally simple.
